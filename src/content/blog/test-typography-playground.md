阅读体验的 80% 来自排版。同样的内容，不同的字体、字号、行距和间距安排，会带来完全不同的阅读感受。这篇文章本身就是一份排版演示。

## 为什么排版重要

**好的排版是不可见的。** 当读者沉浸在内容中时，不会意识到字体的存在。只有当排版糟糕时——字距太挤、行距太密、代码块看不清——读者才会被迫从内容中抽离。

这与 Notion 的设计哲学一致：**不喧宾夺主，但处处讲究。** 每一个间距、每一条分割线、每一种颜色，都有其功能目的。

## 标题层级的变化

### 三级标题的呈现

这是三级标题下的正文。三级标题 `h3` 使用 1.25rem / 600 weight，比二级标题略小但保留足够的视觉权重。

#### 四级标题

四级标题 `h4` 使用与正文相同的字号但加粗，适合作为段落内的小标题。它不会打断阅读流，但提供了清晰的视觉锚点。

## 行内元素的处理

文章中的行内代码 `pip install fastapi` 会以紫色高亮显示，与正文形成区分。<kbd>Cmd</kbd> + <kbd>C</kbd> 这样的键盘快捷键使用键帽样式。**粗体**和*斜体*也有各自的视觉权重。

## 多语言代码并排展示

同一个逻辑，三种语言的实现对比：

```python
# Python
def fibonacci(n):
    a, b = 0, 1
    for _ in range(n):
        yield a
        a, b = b, a + b
```

```javascript
// JavaScript
function* fibonacci(n) {
  let [a, b] = [0, 1];
  for (let i = 0; i < n; i++) {
    yield a;
    [a, b] = [b, a + b];
  }
}
```

```rust
// Rust
fn fibonacci(n: u32) -> Vec<u32> {
    let mut seq = vec![0, 1];
    for i in 2..n as usize {
        seq.push(seq[i - 1] + seq[i - 2]);
    }
    seq
}
```

<div class="callout callout-warning">
<strong>⚠️ 常见误区</strong>
不要为了"好看"而使用过多的字体、颜色和装饰效果。排版的核心目标是清晰传达信息，而非展示设计能力。每增加一个视觉元素，都应该问自己：它帮助读者理解内容了吗？
</div>

---

正文之间的间距遵循倍率关系：段落间距 1rem，标题与正文间距 1.5-2rem，区块间距 1.25rem。一致的节奏感是好的排版的基础。

<div class="callout callout-tip">
<strong>✅ 设计原则</strong>
保持间距的一致性比追求完美更重要。读者适应了某种节奏后，任何偏差都会造成干扰。
</div>

## 结论

这篇排版实验展示了一个原则：**排版是内容的载体，不是装饰。** 好的排版系统会让读者忘记"排版"的存在，只关注内容本身。
